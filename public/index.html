ELVA Notify Service

Internal communication service for OTP and notifications via SMS and Email.

Base URL

https://notify.elvatech.in

Authentication

All endpoints require the following credentials:

  • appId: Application identifier
  • apiKey: Secret key for that application
{
      "appId": "my-app",
      "apiKey": "your-secret-key"
    }

OTP APIs

POST /otp/send

Generates a 6-digit OTP and sends it via SMS.

{
      "appId": "my-app",
      "apiKey": "your-secret-key",
      "phone": "+919876543210"
    }

Response:

{
      "success": true,
      "message": "OTP sent successfully",
      "expiresIn": 300
    }

POST /otp/resend

Resends OTP after invalidating previous one.

{
      "appId": "my-app",
      "apiKey": "your-secret-key",
      "phone": "+919876543210"
    }

POST /otp/verify

Verifies OTP entered by user.

{
      "appId": "my-app",
      "apiKey": "your-secret-key",
      "phone": "+919876543210",
      "otp": "123456"
    }

Response:

{
      "success": true,
      "message": "OTP verified successfully"
    }

Notification API

POST /notify/send

Sends notification message via SMS.

{
      "phone": "9876543210",
      "message": "Your task is approved"
    }

Request Rules

  • Provide either phone OR email (not both)
  • API key required in header
  • OTP expires in 5 minutes

Error Response

{
      "success": false,
      "error": "validation_error",
      "message": "Invalid input"
    }

Rate Limiting

  • 3 OTP requests per minute
  • 10 OTP requests per hour
  • Applies per phone number

Health Check

GET /health

Used to verify service status

Sample CURL

        curl -X POST https://notify.elvatech.in/otp/send \
        -H "Content-Type: application/json" \
        -H "x-api-key: YOUR_API_KEY" \
        -d '{"phone":"9876543210"}'
